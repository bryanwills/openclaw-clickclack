# Message reactions behavior proof

## Real ClickClack run

![A real ClickClack channel showing a sent message with a thumbs-up reaction](https://raw.githubusercontent.com/openclaw/clickclack/main/docs/proof/message-reactions-real-app.png)

The screenshot is generated by `tests/e2e/message-reactions.spec.ts` against a
production ClickClack web bundle served by the Go API. The scenario creates a
workspace and channel through the API, sends a message through the rendered
composer, and adds the thumbs-up reaction through the rendered picker.

The reaction coverage verifies:

- authoritative mutation responses and persisted per-user state
- realtime add/remove updates without a message refetch
- keyboard focus, Escape handling, and `aria-pressed` state
- optimistic rollback without erasing a newer realtime reaction
- a newer realtime event winning over a delayed mutation response
- reaction events surviving stale paginated message responses
- shared reaction state across timeline, thread root, and thread replies
- reaction controls and realtime updates in channel and thread embeds

Store tests cover SQLite and PostgreSQL aggregation, viewer-specific
`reacted_by_me` state, nonce lookups, thread windows, and concurrent PostgreSQL
writers producing ordered authoritative counts.

## Reproduction

```sh
REACTION_PROOF_PATH="$PWD/docs/proof/message-reactions-real-app.png" \
  pnpm exec playwright test \
    tests/e2e/message-reactions.spec.ts \
    tests/e2e/embed-channel.spec.ts \
    tests/e2e/embed-thread.spec.ts
```
